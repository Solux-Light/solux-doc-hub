01 — Colour

Primary Palette — Option A · Steel

Five colours derived from the Steel system. Professional, calm, and tech-adjacent. Blue accent replaces gold entirely — consistent with Solux's B2B solar and engineering positioning. All UI colour decisions reference only these five values and their alpha derivatives.

Ivory
#F7F3EE
Page background · Light surface · Warm neutral
Deep Charcoal
#1C2030
Primary text · Header background · Highest contrast
Rich Navy
#1C3254
Section headers · Sidebar · Primary action surfaces
Steel Blue
#4A6FA5
Links · Active states · Pills · Accent highlights
Mist
#C8D4E3
Borders · Dividers · Table row alt · Subtle fill
Rule: These five values are the only permitted base colours. All other UI values must be derived from them using the alpha token system below. Do not introduce new hues — use opacity to create hierarchy within the palette.

02 — Typography

Type System

Three typeface roles. Sans-serif carries all UI, data, and body. Serif is reserved for section titles and document headers only — never for table cells, labels, or body copy. Mono is for code, hex values, CSS tokens, and technical specs.

Primary — UI & Body
Aa Bb Cc
Helvetica Neue / Arial
font-family: "Helvetica Neue",
Arial, sans-serif
All UI text, body copy, labels, table cells, nav, buttons, tags, pills, dashboards. Default for all document content.
Secondary — Display & Headings
Aa Bb Cc
Georgia
font-family: Georgia,
"Times New Roman", serif
Section titles (h1 only), document headers, style guide display text. Never for table rows, labels, or body paragraphs.
Tertiary — Code & Specs
#4A6FA5
SF Mono / Fira Code
font-family: "SF Mono",
"Fira Code", "Courier New",
monospace
Hex values, CSS variable names, token names, code snippets, API references. Anywhere precision and machine-readability matters.

04 — Spacing

Spacing Scale

Base unit: 8px. All spacing values in the Solux documents should be multiples of 4px (half-unit) or 8px (full unit). Named tokens replace raw pixel values in CSS. Use the smallest step that creates clear visual separation — do not default to large gaps.

Token Value Visualisation Primary Use Example
--sp-1
4px · 0.25rem

Between tag and icon; badge inner vertical padding

Spacing rule: If two elements feel too close, add one step up (e.g. 8px → 12px). If a gap feels excessive, step down. Never use arbitrary values like 7px, 15px, or 22px — always stay on the 4px grid.
