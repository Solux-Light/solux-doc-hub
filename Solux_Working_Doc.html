0. One-Page Summary

Main Problem

Solux already has valuable systems, product expertise, project history, and AI ideas — but the operational workflow is fragmented. The CRM is currently in the build phase and has not yet been deployed to end-users. When it goes live, it must immediately guide daily work, enforce workflow discipline, reduce manual re-entry, and give management clear visibility.

Highest-Impact Focus

The first priority is not to build everything. The first priority is to make the CRM and related workflow tools useful for the most important daily operations:

  1. Lead intake and classification
  2. Sales qualification
  3. Study Lab request handoff
  4. Quotation tracking and follow-up
  5. Owner dashboard and reporting
  6. UAT / usability improvement
  7. AI-assisted support — only after core workflow structure is clean

Recommended Direction

Start with workflow clarity and CRM refinement before adding advanced AI. AI should support classification, missing-information checks, summaries, and drafting — but it should not replace structured CRM rules, required fields, approvals, or human validation.

Phase 1 vs Phase 2

This document covers Phase 1: CRM foundation, workflow clarity, and AI-assisted basics. Phase 2 (Business Analysis & AI Initiatives) builds on this foundation with advanced AI projects, integrations, and automation. AI Initiatives should not start before Phase 1 workflows are stable.

3. Roadmap — Week 1 + 3 Months

This roadmap focuses on the highest-impact changes that make the CRM and workflow system useful, accurate, and ready for adoption. Future initiatives remain visible in the continuation backlog — they are not excluded, they are sequenced.

Week 1
Discovery Sprint
Goal: Get oriented fast. Confirm what exists, what's broken, and what to fix first.

Close working relationship with Mehdi and the teams means discovery does not need a full month. One focused week is enough to lock the plan.

Main Work

  • CRM access and first walkthrough with Mehdi
  • One session each with sales and engineering to confirm handoff pain points
  • Review sample records: lead, quote, Study Lab request, after-sales case
  • Confirm which workflows match Mehdi's files and which differ in practice
  • Lock the problem-impact-fix backlog and priority order

Deliverables

  • CRM audit notes
  • Confirmed workflow map
  • Locked problem-impact-fix backlog
  • Open questions tracker

Solux Input Needed

  • CRM access before Week 1 starts
  • Mehdi walkthrough session — Day 1–2
  • Sales and engineering sessions — Day 3–4
Month 1
CRM Workflow Refinement
Goal: Edit, polish, and improve the CRM based on confirmed workflow problems from Week 1.

Main Work

  • Clean Lead, Account, Contact, Project, and Deal structure
  • Add or refine project qualification fields
  • Define required fields by stage
  • Create Study Request workflow with statuses and handoff checklist
  • Refine sales pipeline and quote stages
  • Add task and follow-up logic
  • Improve naming, labels, field grouping, and screen clarity

Deliverables

  • Improved lead structure
  • Project qualification form
  • Study Request pipeline
  • Quote workflow draft
  • Required-field rules
  • Follow-up task rules
  • Updated CRM change log

Solux Feedback Needed

  • Validate required fields
  • Confirm sales and engineering handoff rules
  • Confirm quote stages
  • 2 business days for small changes · 3 business days for major decisions
Month 2
Automation, Dashboard & AI-Ready Foundation
Goal: Reduce manual work and improve visibility without adding unnecessary complexity.

Main Work

  • Define and implement automation rules
  • Lead assignment and follow-up reminders
  • Study Request alerts
  • Quote follow-up reminders
  • Build owner dashboard
  • Build sales and Study Lab dashboards if possible
  • Define AI-Assisted MVP — only after CRM fields and workflow are stable

Recommended First AI-Assisted Features (in order)

  1. Lead classifier
  2. Missing-information checker
  3. Draft client reply
  4. Owner daily summary
  5. Product direction suggestion — only if product data is structured enough

Deliverables

  • Automation rules live
  • Dashboard setup
  • AI readiness checklist
  • AI-Assisted MVP scope confirmed
  • Cost-control rules
  • Human approval rules defined

Solux Feedback Needed

  • Approve automations
  • Approve dashboard metrics
  • Confirm first AI-assisted use cases
  • Confirm AI budget and API key ownership
Month 3
UAT, Usability Testing, Training & Handover
Goal: Make sure the CRM is not only functional, but obvious and usable for daily work from day one.

Main Work

  • Test workflows with real Solux cases
  • Run usability-focused UAT
  • Evaluate screens using heuristic UX principles
  • Record confusing screens, missing context, unclear statuses, and unnecessary clicks
  • Fix high-friction areas
  • Create usage guide and SOPs
  • Train users — getting adoption right from day one
  • Define next-phase backlog (Phase 2 = AI Initiatives)

Deliverables

  • UAT test cases
  • UX / usability issue log
  • Final CRM workflow updates
  • User guide
  • Admin guide
  • Handover notes
  • Phase 2 continuation backlog

Solux Feedback Needed

  • End-users test real workflows
  • Feedback within 3 business days
  • Final sign-off or continuation backlog confirmation

4. UAT / Usability Checklist + Heuristic Evaluation

Functional UAT

For each workflow, check:

QuestionPass / FailNotes
Can the user complete the task from start to finish?
Does the saved data appear correctly?
Does the status update correctly?
Are required fields enforced?
Are tasks/reminders created correctly?
Does the right user receive the right notification?
Can the owner see the result in the dashboard?

UX / Usability Testing

For every screen, ask:

QuestionNotes
Would a brand-new employee know what to do?
Is the main action obvious?
Is there too much information?
Is important information missing?
Are buttons in the right place?
Is the visual hierarchy clear?
Does the user know what happened after clicking?
Is the next step obvious?
Are there too many clicks?
Are mandatory fields obvious?
Are statuses easy to understand?
Are there places where users may make mistakes?
Are reminders or contextual hints missing?

Heuristic Evaluation

UX PrincipleWhat To Check
ClarityLabels, buttons, statuses, and forms are easy to understand
EfficiencyUsers can complete common tasks with minimal clicks
FeedbackSystem clearly shows what happened after save, submit, or status change
Error preventionRequired fields, validations, and warnings prevent bad data
ConsistencySimilar screens use similar wording, layout, and actions
VisibilityImportant statuses, next steps, and owners are visible
Recognition over memoryUsers should not need to remember what to do next
RecoveryUsers can fix mistakes or understand why something failed
PleasantnessScreen feels clean, balanced, and not stressful for daily use

7. AI Should Not Be First Everywhere

AI Readiness Rule

AI should only be added after the workflow and source data are clean. Before any AI feature, Solux needs:

  1. Clean CRM fields
  2. Clear pipeline stages
  3. Required fields by workflow stage
  4. Product data source
  5. Human approval rules
  6. Dashboard/reporting logic
  7. End-user adoption of the basic CRM flow

Good First AI-Assisted Use Cases

AI Use CaseSafe?Why
Lead classificationYesHelps categorise faster
Missing-information checkerYesReduces incomplete handoff
Draft client replyYesHuman reviews before sending
Owner daily summaryYesSummarises existing CRM data
Product direction suggestionMediumNeeds clean product data first
Quote approvalNoPricing/margin must stay human-controlled
Final engineering decisionNoTechnical validation must stay with engineering
Rule

AI can suggest, summarise, classify, and draft. AI should not approve pricing, finalise product selection, override engineering decisions, or update critical CRM data without human confirmation.

9. Klairs Working Notes

My Working Position

The goal is not only to improve the CRM interface. The real goal is to help Solux turn fragmented manual operations into a more reliable business workflow system.

The CRM is the operational base, but the larger improvement includes lead intake, qualification, Study Lab handoff, quotation, follow-up, dashboards, automation, product knowledge, AI support, and future system integrations.

This document should help me track what exists, what is broken, what should be changed first, what decisions are needed, and what should be sequenced for continuation.

My Current Assumptions

AssumptionConfidenceHow To Validate
The CRM is currently in the build phase and has not yet been deployed to end-users (sales, engineering, support)HighConfirm current build status and go-live timeline with Mehdi
The biggest early value is workflow clarity, not AIHighConfirm with sales and engineering pain points
Study Lab handoff is one of the highest-ROI areasHighReview real Study Lab requests
Quote tracking and follow-up are likely leaking revenueHighReview recent quote process and follow-up history
AI can help, but only after CRM data and workflow are structuredHighCheck field quality, product data, and CRM usage plan
Future initiatives are necessary, but should be sequenced carefullyHighBuild continuation roadmap after first implementation cycle

My Guiding Principles

1

Do not build randomly.

2

Do not add AI before the workflow is clear.

3

Do not overcomplicate the CRM.

4

Make every screen obvious for a new employee.

5

Prioritise the workflows that save the most time or prevent the most revenue leakage.

6

Turn every vague problem into a specific CRM change, automation rule, dashboard, checklist, or decision.

7

Keep future initiatives visible, but sequence them based on readiness and ROI.

8

Test with real Solux cases, not imaginary examples.

9

Always separate confirmed facts from assumptions.

10

Make the system useful enough that the team actually wants to use it.
