CRM: Reflect the 7-Office Structure

Create 6 territory records in the CRM (one per active sales office). Every Lead and Deal must be tagged with a territory field. Assignment rules: country → auto-assign to correct office. Dubai = Gulf countries; Dakar = West Africa; Riyadh = Saudi/GCC; Vietnam = SEA; France = Europe + Morocco.

2.2 Sales Qualification

📄 Current State

Problem

No structured project qualification form exists between Sales and Engineering. When a lead is ready to request a study, the salesperson describes the project via email or WhatsApp. The engineering team frequently needs to ask follow-up questions because key data is missing (GPS location, road type, luminaire count, operating hours, grid availability, etc.).

This back-and-forth adds days or weeks to every project cycle and creates frustration on both sides.

Source: CONSULT · MEHDI Lighting Study Platform file
💡 Claude's Suggestions → See P02 Study Platform
CRM EDIT Qualification Checklist Stage Gate

Add a stage gate in the CRM Deal pipeline: a Deal cannot move from "Qualified" to "Study Requested" unless all of these fields are filled: project_location (GPS or city), road_type, luminaire_qty, operating_hours_per_night, grid_available (yes/no), target_delivery_date, client_budget_range.

AI LAYER Missing Info Checker

When a salesperson tries to advance the deal, AI checks the record completeness and blocks with a specific message: "This deal is missing: GPS coordinates and operating hours. Please complete before requesting the study." No vague error — specific fields named.

AUTOMATE Qualification Form for Partners

Build a simple web form (linked from the Study Platform) that partners can fill themselves. Answers flow directly into a new CRM Deal + Study Request without sales rep involvement. Partners get their own qualified entry point.

INTEGRATE Automate Steps 4–6

Steps 4–6 are pure data re-entry. AI LAYER Similar Study Recommender

Before starting a new study, the platform searches ~600 past studies for similar location + product combinations and shows the 3 closest matches. Engineer uses them as references. Estimated time saving: 30–50% per study.

AUTOMATE Study → CRM Callback

When engineer marks study "Delivered" → Study Platform fires webhook back to CRM → Deal advances to "Study Complete" stage + PDF report attached automatically. Sales rep is notified.

📄 Current State

Quotations are likely prepared manually in Excel or Word, sent as PDF by email. No version control — multiple versions of the same quote may exist with no clear "current" version. No approval workflow. No automated follow-up when a quote goes unanswered.

Source: INFERRED — absence of quote module across all sources
💡 Claude's Suggestions
CRM EDIT Quote Object

Add a Quote object linked to each Deal. Fields: quote_number (auto-generated), version (v1, v2…), products (from Product Hub), quantities, unit_prices, discount_pct, total_value, currency, valid_until, status (Draft / Sent / Accepted / Rejected / Expired).

CRM EDIT Approval Rule

Quotes above a certain value (e.g., €50K) require owner approval before sending. CRM blocks the "Send" action until the owner marks it "Approved." One click in the CRM — no email chain for internal approval.

AUTOMATE Follow-up Reminder

If a sent quote has had no response in 7 days → automatic task created for the sales rep: "Follow up on Quote #[number] sent to [client]." If no response in 14 days → second reminder. This stops revenue leaking through forgotten quotes.

AI LAYER Price Intelligence

When creating a new quote, AI queries the Commercial Intelligence Database (P07) for historical quotes to the same country/product combination → suggests a price range based on what has won and lost in the past. Informs pricing decisions without manual searching.

2.6 After-Sales & Support

📄 Current State

Support Channel

After-sales cases handled via WhatsApp and email. No case management system. No diagnosis history per installation. Warranty status must be checked manually.

What Solux Has (unused for CRM)

Solux Bridge — live IoT monitoring system on installed products. Provides real-time:

  • Battery state of charge
  • Solar production history
  • Temperature readings
  • Fault alerts
  • Remote wireless diagnosis tool

Despite having live IoT data, after-sales cases are not linked to Solux Bridge readings. An engineer responding to a complaint has no visibility of the installation's recent history.

Source: CONSULT WhatsApp handling · WEB-TECH Solux Bridge specs
💡 Claude's Suggestions
CRM EDIT After-Sales Case Object

Add a "Case" object to the CRM linked to the original Deal/Installation. Fields: installation_id, product_serial, issue_description, warranty_status (in-warranty / expired), case_status (Open / In Diagnosis / Resolved), resolution_notes, parts_sent.

INTEGRATE Solux Bridge → CRM Case

When a new After-Sales Case is opened, auto-pull the last 30 days of Solux Bridge data for that installation's product ID → attach to the Case record. The engineer diagnosing the problem immediately sees: battery % trend, solar production anomalies, temperature spikes, fault alert log. Remote diagnosis before any site visit.

AUTOMATE Proactive Fault Detection

Future state: Solux Bridge fault alerts → if severity = critical → auto-create a CRM After-Sales Case before the customer even calls. Transforms support from reactive to proactive. Customer receives an email: "We detected an issue with your installation — our team is already investigating."

AUTOMATE WhatsApp Intake

Short term (before full case system): set up a WhatsApp Business number connected to n8n. When a customer sends a support message → n8n creates a CRM Case with the WhatsApp transcript. Better than nothing while the full case module is built.

Integration Priority Map

INTEGRATE The Central Hub: CRM

The CRM must become the data hub. Everything flows into and out of it. Integration order: (1) Joomla form → CRM, (2) JivoChat → CRM, (3) Study Platform ↔ CRM, (4) Tender Tool → CRM, (5) Mailchimp/Brevo ↔ CRM, (6) Solux Bridge → CRM cases.

AUTOMATE n8n as Integration Middleware

Use n8n (self-hosted, free) as the glue between all systems. It handles webhooks, scheduled jobs, API calls, and CRM writes. One tool manages all integrations — no spaghetti of individual connections. Can run on a €5/month VPS.

CRM EDIT ERP: Don't Buy One Yet

The commercial operation (lead-to-quote, study management, after-sales) is fully addressable through the CRM with AI. ERP should only be considered when manufacturing scheduling, inventory, and financial reporting outgrow spreadsheets. That's a 12–18 month decision, not today.

AUTOMATE Week 1 Priority: Don't Build Yet

The instinct is to start building immediately. Resist. The most valuable Week 1 action is access to the actual CRM and Study Platform — read-only. This is the fastest visible win. The Joomla form already has structured categories. Adding a webhook is 2–4 hours of work. From Day 1 of going live, every website form submission appears in the CRM automatically. Owner sees the impact immediately — builds confidence in the whole programme.

Set it up to send a Slack or email notification with each classification: "New lead from Vietnam classified as: Pro Line enquiry, urgency 4/5, assigned to [rep]." Make the AI visible — don't let it work silently or nobody believes it's running.
